Project Name: InsurTech Assistant
Role: Expert Full-Stack Developer & AI Architect

Core Mission: Develop a dual-agent insurance automation platform designed to minimize manual labor for adjusters by handling data collection and FAQs, requiring human intervention only for final verification or complex escalations.

1. The Two-Agent Architecture:

Agent A: Claims Specialist: A conversational interface that follows a structured "First Notice of Loss" (FNOL) flow. It must collect: Policyholder Name, Policy ID, Date/Time of Incident, Type of Claim (Auto/Home/Life), and a detailed description.

Agent B: Inquiry Specialist: A RAG-powered (Retrieval-Augmented Generation) bot that answers policy FAQs. It should use a mock knowledge base (JSON/PDF) to explain coverage limits, deductibles, and process steps.

2. The Human-in-the-Loop (HITL) Workflow:

Verification Queue: Claims generated by the bot are saved as "Drafts" in the PostgreSQL database.

Adjuster Dashboard: Create a password-protected view /admin where adjusters can see a list of Draft Claims.

The "Verify" Action: Adjusters must be able to review the bot's summary, edit any fields, and click a "Verify & Submit" button to finalize the claim.

Live Handoff: Include a "Request Human" button in both chat interfaces that alerts the dashboard, allowing an adjuster to join the chat session.

3. UI/UX Requirements:

Branding: Clean, minimalist, and trustworthy. Primary colors: Navy Blue and Professional Grey.

User Interface: A landing page with two clear cards: "I need to file a claim" and "I have a policy question."

Mobile-First: Ensure the chat interface is fully responsive for customers filing claims from their phones at the scene of an incident.

4. Technical Stack:

Framework: Python (Flask or FastAPI) for the backend.

Database: Replit PostgreSQL for storing chat logs and claim records.

AI Integration: Use OpenAI's GPT-4o or Claude 3.5 Sonnet API via Replit's Secret environment variables.

Key Features I’ve Optimized for the Adjuster:
The "Prep-Work" Logic: The Claims bot doesn't just record text; it formats a Summary Report. Instead of an adjuster reading a transcript, they read a 3-bullet point summary generated by the AI.

The Sentiment Trigger: If a user in the Inquiry chat uses keywords like "legal action," "lawyer," or "complaint," the app should automatically flag that session as High Priority on the Adjuster's dashboard.

Audit Trail: Every claim will show "Collected by AI, Verified by [Adjuster Name]," ensuring accountability.
